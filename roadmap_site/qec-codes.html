1 · Stabilizer formalism recap

A stabilizer code on $n$ physical qubits is defined by an Abelian subgroup $\mathcal{S}$ of the Pauli group, with $-I \notin \mathcal{S}$. The codespace is

$$ \mathcal{C} = \{ |\psi\rangle : S|\psi\rangle = |\psi\rangle \;\forall S\in \mathcal{S} \}. $$

If $\mathcal{S}$ has $n - k$ independent generators, the codespace encodes $k$ logical qubits. Logical Pauli operators $\bar X, \bar Z$ commute with $\mathcal{S}$ but are not in it.

The art of QEC is choosing $\mathcal{S}$ so that any low-weight error on the physical qubits anti-commutes with at least one generator — measuring it raises a non-trivial syndrome that pinpoints the error.

A stabilizer code has distance $d$ if the minimum-weight non-trivial logical operator has weight $d$. It can correct any error of weight $\lfloor (d-1)/2 \rfloor$. The two codes most relevant to our roadmap (surface and color) are both topological: stabilizers are geometrically local on a 2D lattice.

2 · Surface code

The surface code places physical qubits on the edges of a 2D lattice. Each vertex carries an $X$-stabilizer (product of $X$ over incident edges); each plaquette carries a $Z$-stabilizer.

  • Distance-$d$ code uses $d^2$ data qubits (rotated layout) and ~$d^2$ ancillas. $d=3$ → 9 data + ~9 ancilla.
  • Threshold $\approx 1\%$ per physical operation under standard noise models — the highest of any known topological code.
  • Logical operators $\bar Z$ runs vertically across rows; $\bar X$ runs horizontally across columns.
  • Lattice surgery implements two-qubit logical gates by merging and splitting patches.
Why surface code first

Highest threshold, simplest connectivity (nearest-neighbour on a square lattice), and a mature decoder (MWPM / Union-Find). Already demonstrated below threshold by Google & ETH Zurich (2022–2024). Our M8 milestone targets distance-3 (smallest fault-tolerant) for benchmarking against Stim.

3 · Color codes

Color codes live on a 3-coloured 2D lattice. Every face is a stabilizer of both $X^{\otimes n}$ and $Z^{\otimes n}$ types, giving a $[[n,1,d]]$ code with the same logical-information ratio as the surface code but with transversal Clifford gates.

LatticeFaces (stabilizers)ThresholdProsCons
6.6.6 (triangular)Hexagons~0.3–0.5%Transversal Clifford, low overheadWeight-6 stabilizers (longer syndrome circuits)
4.8.8 (square–octagon)Squares + octagons~0.1%Transversal CliffordWeight-8 stabilizers; harder decoder
When colour beats surface

Color codes implement the entire Clifford group transversally — every logical gate is a single round of physical gates with no $T$-state distillation overhead for the Clifford part. This dramatically reduces magic-state-injection rates. The cost is more complex syndrome circuits and lower threshold.

4 · Syndrome extraction

Each stabilizer generator is measured by an ancilla qubit running a "merge–CNOT–measure" pattern. A weight-4 $Z$-stabilizer needs:

sequenceDiagram
  participant A as Ancilla |0⟩
  participant D1 as Data q1
  participant D2 as Data q2
  participant D3 as Data q3
  participant D4 as Data q4
  A->>D1: CNOT
  A->>D2: CNOT
  A->>D3: CNOT
  A->>D4: CNOT
  Note right of A: Measure Z on ancilla
= eigenvalue of Z⊗Z⊗Z⊗Z

For $X$-stabilizers, the ancilla begins in $|+\rangle$ and is measured in the $X$-basis (equivalently surrounded by Hadamards). Each cycle requires:

  1. Reset all ancillas.
  2. Hadamards (for $X$-stabilizers).
  3. Four CNOTs (in a schedule that avoids collisions on data qubits).
  4. Hadamards back.
  5. Dispersive readout of every ancilla.
  6. Classical decoding of the syndrome.

In a molecular-spin / cQED hybrid, the natural choice is:

  • Data qubits are molecular spins (long coherence).
  • Ancilla qubits are transmons (fast initialisation, fast dispersive readout).
  • CNOTs are cross-resonance or cavity-mediated $J_\text{eff}$ between spin and transmon.
  • Readout uses the shared bus cavity dispersively coupled to the transmon.

6 · Simulating logical qubits in LindbladTTN (M8, shipped)

The v0.3.0 stabilizer toolkit gives you the code structure, the abstract syndrome-extraction circuit, and the MWPM decoder. Compilation to physical pulses + full Lindblad simulation of the syndrome rounds is the natural next step (will lean on PS2 + the multi-child state initialiser):

from lindblad_ttn.qec import (
    RepetitionCode, SurfaceCode, ColorCode488, PauliString,
)
from lindblad_ttn.qec.decoders import mwpm_decode_syndrome

# ---------- 1) Stabilizer codes ----------
rep   = RepetitionCode(N=5)              # 5 data qubits, distance 5
surf  = SurfaceCode(distance=3)          # rotated d=3
color = ColorCode488()                   # 7-qubit Steane

# Each code knows its generators and logical operators.
surf.verify_commutation()                # asserts all stabilizers commute
print(f"surface  d={surf.distance},  n_data={surf.n_data}, "
      f"|stabs|={len(surf.generators())}")
print(f"Xbar weight = {surf.logical('X').weight()}")  # 3 for d=3

# ---------- 2) Abstract syndrome circuit ----------
gates = surf.syndrome_circuit()
# [(name, qubits), ...]  with name in {'RESET', 'H', 'CNOT', 'MZ', ...}

# ---------- 3) MWPM decoding ----------
import numpy as np
# `H` is the parity-check matrix (n_stabs × n_errors); `syn` is a batch of
# observed syndromes from a noisy simulation.
syn = np.array([[1, 0, 1, 1, 0, 0]], dtype=np.uint8)
H_pcm = np.eye(6, dtype=np.uint8)
correction = mwpm_decode_syndrome(syn, H_pcm)  # falls back to brute-force if
                                               # PyMatching is not installed
What ships now vs. what's next

v0.3.0 ships: code structure, generators, logical operators, abstract syndrome circuits, MWPM decoder wrapper. Next: compiling the abstract circuit into pulse sequences via the M6 pulse library, and end-to-end Lindblad syndrome simulation with the cavity-centred TTN topology — both unlocked once the 2-site TDVP (PS2) and the multi-child state initialiser land.

Reality check

A full distance-3 syndrome simulation has $\sim 17$ qubits + 2 cavities = 19 sites, each $d=2$–$8$. Even with TTN compression this is a flagship-scale simulation — measure twice, run once. The benchmark suite (M10) tracks where the bond dimension blows up.
