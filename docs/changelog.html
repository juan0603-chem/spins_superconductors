Release notes

v0.3.0 — Hybrid-platform release

Per-site dimensions, bosonic modes, higher-spin sites, Schrieffer–Wolff tools, a pulse library, a stabilizer-code toolkit, the QuTiP-matched Bell-parity test, and the PS1 2× rate-bug fix — everything from milestones M1 through M10 in one release.

What's new

Per-site dimensions (M1)

New constructor surface on LindbladTTN accepts a list of Site objects and product-form H_terms / L_terms dictionaries. Each site carries its own local dimension, so a single tree can mix spin-1/2 qubits, spin-S sites, and truncated bosonic modes. The legacy spin-1/2 path (H0=, L_ops=, n_sites=) is untouched and continues to pass byte-for-byte. 

from lindblad_ttn import LindbladTTN
from lindblad_ttn.sites import spin_half_site, boson_site

q = spin_half_site("q0")
c = boson_site(8, "c0")

solver = LindbladTTN(
    sites    = [q, c],
    H_terms  = [(0.5 * omega_q, {q.name: q.sz}),
                (omega_c,        {c.name: c.adag @ c.a}),
                (g,              {q.name: q.sp, c.name: c.a}),
                (g,              {q.name: q.sm, c.name: c.adag})],
    L_terms  = [(kappa, {c.name: c.a})],
    bond_dim=16, topology="train", strategy="ps1",
)

Bosonic and spin-S sites (M2, M3)

BosonSite(N_cut) exposes a, adag, n, x, p, plus a kerr(K) helper and Fock / coherent-state vectors. SpinSite(S) exposes Sx, Sy, Sz, Sp, Sm, and Stevens operators $O_k^q$ for the dominant crystal-field terms used in molecular magnetism.

Hamiltonian templates (M4)

Pre-built bundles that return (sites, H_terms, L_terms): transmon, fluxonium, bare_lc, vanadyl_spin (with optional 51V nuclear spin), lanthanide_smm, jaynes_cummings, tavis_cummings, dispersive_readout, and a merge() helper for stacking them.

Effective-Hamiltonian tools (M5)

schrieffer_wolff(H0, V, order=4) for block-diagonalising perturbations; dispersive_shift with $\langle\sigma_z\rangle$-based eigenstate labelling; magnus_average at orders 1 and 2.

Pulse-shape library (M6)

gaussian, drag (returns the $(f_I, f_Q)$ pair for transmon leakage suppression), square_rise, cosine_drive, plus sequence and integrate_pulse helpers. Drop directly into drives_nd.

Steady-state & energy levels (M7)

steady_state_dense(site_dims, dof_names, H_terms, L_terms) builds the full Liouvillian as a dense matrix and returns the eigenvector with smallest |λ|, normalised to Tr(ρ) = 1. energy_levels_dense(H, k) wraps scipy.linalg.eigh.

Stabilizer toolkit (M8)

PauliString sparse-support class; StabilizerCode base; concrete RepetitionCode, rotated SurfaceCode (odd distance), and ColorCode488 (Steane); abstract syndrome-extraction circuits as (RESET, H, CNOT, MZ) gate lists; MWPM decoder wrapping PyMatching with a brute-force fallback for environments without PyMatching installed.

Cavity-centred topology (M9)

LindbladFrameFactory.cavity_centered(cavity_dofs, spin_dofs) builds a star/spine with bosonic Nodes at the centre and spins on leaves — the natural connectivity for cavity-mediated all-to-all Hamiltonians.

Benchmark suite (M10)

Four new test modules under benchmarks/: site operators, effective Hamiltonians, QEC codes, and JC vs QuTiP (both lossless and with cavity decay). Run with pytest benchmarks/ -q.

Bug fixes

PS1 2× rate bug (≥ 2 sites)

The original _ps1_step visited each link four times per integration step — the round-trip link order from frame.node_link_visitor already encodes both directions of each link, but the outer "forward sweep + reversed sweep" pattern applied each sweep at dt/2, double-counting every node evolution. Result: dynamics ran at exactly 2× the expected rate for any multi-site simulation. Documented as xfail in test_qutip_parity::test_bell_parity_qutip.

Fixed by replacing the round-trip link order with a single-direction DFS link order (parent → child once), then applying it forward at +dt/2 and reversed at +dt/2. Each node is now evolved at dt total per step, restoring Strang-symmetric accuracy. See propagation/tdvp.py:_ps1_step and :_directed_link_order.

Missing env_above[target]

The two-pass environment builder in effective_h.py was skipping env_above computation when the prospective child was the target node, so H_eff at non-root targets missed the parent-side contribution. Fixed in the new unified _compute_envs by removing the if nbr is target: continue guard.

End-to-end example

examples/hybrid_vanadyl_transmon.py stitches the whole stack together: a vanadyl-style spin coupled to a transmon via a shared bus, with DRAG drive on the transmon, T₁ + κ dissipation, and PS1 integration. 

from lindblad_ttn import LindbladTTN
from lindblad_ttn.control import drag
from lindblad_ttn.sites import boson_site, spin_half_site

q_van = spin_half_site("vo")
c_bus = boson_site(6, "cav")
q_tx  = spin_half_site("tx")

H_terms = [
    (0.5 * omega_van, {q_van.name: q_van.sz}),
    (omega_cav,       {c_bus.name: c_bus.adag @ c_bus.a}),
    (0.5 * omega_tx,  {q_tx.name: q_tx.sz}),
    (g_van, {q_van.name: q_van.sp, c_bus.name: c_bus.a}),
    (g_van, {q_van.name: q_van.sm, c_bus.name: c_bus.adag}),
    (g_tx,  {q_tx.name: q_tx.sp, c_bus.name: c_bus.a}),
    (g_tx,  {q_tx.name: q_tx.sm, c_bus.name: c_bus.adag}),
]
L_terms = [(kappa, {c_bus.name: c_bus.a}),
           (gamma_van, {q_van.name: q_van.sm}),
           (gamma_tx,  {q_tx.name: q_tx.sm})]

f_I, _ = drag(amp=0.05, t0=50.0, sigma=10.0, anharm=-0.3, beta=0.5)
drives_nd = [(f_I, [(1.0, {q_tx.name: q_tx.sx})])]

solver = LindbladTTN(
    sites=[q_van, c_bus, q_tx],
    H_terms=H_terms, L_terms=L_terms, drives_nd=drives_nd,
    bond_dim=16, topology="train", strategy="ps1",
)

Compatibility

  • Python 3.9+ (uses from __future__ import annotations everywhere).
  • PyTorch ≥ 2.0, torchdiffeq ≥ 0.2, numpy ≥ 1.24, scipy ≥ 1.10.
  • Optional: qutip (for the parity tests), pymatching (MWPM decoder; falls back to brute-force).
  • No breaking changes to the legacy spin-1/2 API. All existing scripts continue to work.
