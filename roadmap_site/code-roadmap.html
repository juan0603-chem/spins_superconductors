Code roadmap

Ten dependency-ordered milestones that grew LindbladTTN from a spin-1/2 Lindblad solver into a hybrid-platform simulator with stabilizer-code support. All ten are shipped.

Status

M1–M10 are implemented, the pre-existing PS1 2× rate bug for ≥2-site simulations is fixed, and the QuTiP-parity tests for the dispersively coupled Bell-state circuit pass at the 1.6 × 10⁻⁴ level. See the changelog for the full diff and benchmark numbers.

Before / after

Before M1, the library handled N spin-1/2 qubits in interleaved Liouville space. Every operator was Pauli-decomposed and mapped to 4×4 local superoperators. Three things were hard-coded at the $d=2$ level:

  • pauli_decompose in physics/liouvillian.py assumed $d=2$ at every site.
  • local_dim = 4 appeared in propagation/effective_h.py, propagation/tdvp.py and solver.py.
  • End nodes in core/graph.py had no explicit physical dimension attribute.

M1 generalised the decomposition by adding a parallel physics/liouvillian_nd.py builder that accepts pre-tensored single-site operators (no Pauli decomposition, no $O(4^N)$ scan). Each site's local dimension is carried by its Site object, and the solver threads site_dims through model initialisation, observables, and the row-major ↔ interleaved permutation. The legacy spin-1/2 path is unchanged.

Pre-existing PS1 bug, fixed

PS1 was processing each link four times per step (round-trip link order applied in both forward and backward sweep), causing a documented 2× rate bug for ≥2-site simulations. Fixed by switching to a single-direction symmetric sweep in propagation/tdvp.py:_ps1_step; the previously xfail-marked Bell-state QuTiP-parity test now passes.

Milestones

M1

Per-site local dimensions & generic operator basis

large ✓ shipped

Replaced the Pauli-only operator basis with a per-site SoP builder that takes pre-tensored single-site operators directly — no Pauli decomposition, no $O(4^N)$ scan. Existing $d=2$ behaviour is bit-for-bit identical (all legacy tests pass unchanged).

Files added
physics/liouville_nd.py, physics/liouvillian_nd.py, sites/base.py
Files updated
solver.py (new sites=/H_terms=/L_terms= path), physics/frame_factory.py (DOF naming), propagation/effective_h.py (env_above[target] fix)
API
from lindblad_ttn import LindbladTTN
from lindblad_ttn.sites import spin_half_site, boson_site

q = spin_half_site("q0")
c = boson_site(8, "c0")

solver = LindbladTTN(
    sites    = [q, c],
    H_terms  = [(omega_q/2, {q.name: q.sz}),
                (omega_c,   {c.name: c.adag @ c.a}),
                (g,         {q.name: q.sp, c.name: c.a}),
                (g,         {q.name: q.sm, c.name: c.adag})],
    L_terms  = [(kappa, {c.name: c.a})],
    bond_dim=16, topology="train", strategy="ps1",
)
Validated by
JC vacuum-Rabi vs QuTiP to 5 × 10⁻³ (benchmarks/test_jc_vs_qutip.py); all legacy tests still pass.

depends on: nothing · unblocks: M2–M10

M2

Bosonic site with Fock truncation

medium ✓ shipped

Harmonic-oscillator site with cutoff $N_\text{cut}$. Exposes ladder operators, number, quadrature, and a Kerr helper. Coherent and Fock state constructors included.

Files added
sites/boson.py, registered in sites/__init__.py
API
from lindblad_ttn.sites import boson_site

c = boson_site(N_cut=8, name="c0")
# operators: c.a, c.adag, c.n, c.x, c.p
H_kerr = -0.5 * K * c.adag @ c.adag @ c.a @ c.a
# state helpers: c.fock_state(2), c.coherent_state(alpha=1.5)
Validated by
benchmarks/test_sites.py::test_boson_* (ladder relations, number-op eigenvalues).

depends on: M1 · unblocks: M4, M9, M10

M3

Spin-S site & Stevens operators

medium ✓ shipped

Higher-spin sites $S \in \{1/2, 1, 3/2, 2, 7/2, \ldots\}$ with $S_x, S_y, S_z$, $S_\pm$ and Stevens operators $O_k^q$ for $(k, q) \in \{(2,0), (2,\pm 2), (4,0), (4,2), (4,4), (6,0), (6,6)\}$.

Files added
sites/spin.py
API
from lindblad_ttn.sites import spin_site, stevens_operator

vo  = spin_site(0.5, "vo")           # electron S=1/2
V51 = spin_site(3.5, "I_V51")        # nuclear I=7/2
# operators: vo.Sx, vo.Sy, vo.Sz, vo.Sp, vo.Sm
# crystal-field: vo.stevens(2, 0)  or  stevens_operator(2, 0, S=2)
Validated by
benchmarks/test_sites.py: $[S_x,S_y]=iS_z$ for $S\in\{1/2, 1, 3/2, 2\}$; $S^2$ eigenvalues = $S(S+1)$; $O_2^0$ matches $3m^2 - S(S+1)$.

depends on: M1 · unblocks: M4, M10

M4

Hamiltonian template library

medium ✓ shipped

Pre-built constructors for the hybrid platforms referenced on pages 2–6. Each returns a triple (sites, H_terms, L_terms) consumable directly by LindbladTTN.

Files added
templates.py
API
from lindblad_ttn import LindbladTTN
from lindblad_ttn.templates import (
    transmon, fluxonium, bare_lc,
    vanadyl_spin, lanthanide_smm,
    jaynes_cummings, tavis_cummings, dispersive_readout,
    merge,
)

sites, H, L = jaynes_cummings(omega_q=1.0, omega_c=1.0, g=0.1,
                              N_cut=8, kappa=0.01)

                     bond_dim=16, topology="train")
Validated by
benchmarks/test_jc_vs_qutip.py — JC + cavity decay matches QuTiP to ~5 × 10⁻³ on $\langle\sigma_z\rangle$ and $\langle n\rangle$.

depends on: M2, M3 · unblocks: M5, M8, M10

M5

Effective Hamiltonian tools (Schrieffer–Wolff, Magnus)

medium ✓ shipped

Schrieffer–Wolff transformation up to 4th order via a numerical generator $[H_0, S^{(1)}] = V_\text{od}$; Magnus expansion (orders 1 and 2); dispersive-shift extractor that auto-labels eigenstates by $\langle\sigma_z\rangle$.

Files added
effective.py
API
from lindblad_ttn.effective import (
    schrieffer_wolff, dispersive_shift, magnus_average,
)

H_eff = schrieffer_wolff(H0, V, projector_A=PA, order=4)
chi   = dispersive_shift(H_qc, qubit_dim=2, cavity_dim=8, n_max=1)
H_bar = magnus_average(H_t, period=T, n_points=64, order=2)
Validated by
benchmarks/test_effective.py — $\chi$ within 10% of $g^2/\Delta$ in the dispersive regime; SW 2nd order block-diagonalises a 2-level toy; Magnus(1) of $\cos(\omega t)\sigma_x$ averages to zero.

depends on: M4 · unblocks: M8, M10

M6

Pulse-shape library

small ✓ shipped

Named building blocks for time-dependent control: constant, Gaussian, DRAG (returns $(f_I, f_Q)$), square-with-rise, cosine carrier, and a sequence combiner. Each callable plugs directly into drives / drives_nd.

M8

Stabilizer toolkit & codes

large ✓ shipped

Stabilizer formalism with sparse Pauli strings; repetition, rotated surface (odd distance), and 4.8.8 colour-code (Steane) implementations; abstract syndrome-circuit generator (RESET / H / CNOT / MZ); MWPM decoder wrapping PyMatching with a brute-force fallback.

Files added
qec/__init__.py, qec/stabilizers.py, qec/codes/surface.py, qec/codes/color.py, qec/decoders/mwpm.py
API
from lindblad_ttn.qec import (
    RepetitionCode, SurfaceCode, ColorCode488, PauliString,
)
from lindblad_ttn.qec.decoders import mwpm_decode_syndrome

code.verify_commutation()

Xbar  = code.logical("X")             # PauliString
Validated by
benchmarks/test_qec.py — all generators pairwise commute; logical weights match code distances; syndrome circuits include RESET, CNOT, MZ.

depends on: M4, M5, M7 · unblocks: M10

M9

Cavity-centred tree topology

medium ✓ shipped (factory)

Topology builder that puts the bosonic node(s) at the centre and spin sites at the leaves. Matches the natural Hamiltonian connectivity of cavity-mediated systems. Currently usable through LindbladFrameFactory directly; full state-initialiser support for multi-child interior nodes needs the 2-site (PS2) integrator.

Files updated
physics/frame_factory.py (cavity_centered method + named DOFs)
API
from lindblad_ttn.physics.frame_factory import LindbladFrameFactory

factory = LindbladFrameFactory(dof_names=["c0", "q0", "q1", "q2"])
frame, root, ends = factory.cavity_centered(
    cavity_dofs=["c0"], spin_dofs=["q0", "q1", "q2"],
)
Caveat
The sequential-SVD state initialiser in solver.py only supports trees where every interior node has at most one child; for cavity-centred trees with branching at the cavity Node, use topology="train" until PS2 lands.

depends on: M2 · unblocks: M10

Known limitations after v0.3.0

Limits worth documenting
  • Fixed-rank TDVP1. Both PS1 and VMF hold the bond dimension at the rank set by the initial-state SVD; rank-1 initial states (pure product) cannot grow entanglement during evolution. Workaround: pad bond dim with a small thermal admixture, or move to PS2 (2-site).
  • Cavity-centred state init. The sequential-SVD initialiser only handles trees where each interior node has ≤ 1 child. M9's cavity_centered factory is shipped, but for branching at interior nodes use train until a multi-child initialiser lands.
  • Magnetic-dipole coupling is genuinely small for molecular spins (~1–10 kHz). Flux-mediated and strain-mediated alternatives are flagged on the coupling page.
  • Windows-friendly MWPM. PyMatching is the only well-behaved dependency; the M8 decoder falls back to brute-force enumeration when PyMatching is absent.
  • Bond-dimension blow-up at strong coupling. Past $g \gtrsim 0.1\,\omega$ the dispersive picture fails and the tree compression degrades; will revisit when PS2 / 2-site TDVP lands.

Next milestones (post v0.3.0)

The shipped M1–M10 cover the planning roadmap. The natural follow-up work, in priority order:

  • PS2 (2-site TDVP). Adds rank growth during evolution — fixes the rank-1-initial-state limitation and makes long-time hybrid-platform simulations practical.
  • General multi-child state initialiser. Required to fully unlock the M9 cavity_centered topology; built on top of PS2's bond-expansion machinery.
  • Stim integration for M8. Compare the syndrome-extraction simulator to Stim on a distance-3 surface code over 10⁴ shots.
  • TTN-based steady-state. Krylov/Arnoldi of the Liouvillian acting on a TTN, for $N \gtrsim 12$ where the dense M7 path runs out of memory.

Open problems & stretch goals

  • Lindbladian Krylov methods for ultra-stiff dissipative dynamics (e.g. fast cavity decay during slow spin gates).
  • Hybrid TTN + Monte-Carlo wavefunction for very long syndrome rounds where the tree compression saturates.
  • Differentiable Liouvillian for end-to-end pulse optimisation (drop-in for GRAPE / Krotov).
  • GPU-accelerated MWPM decoder (PyMatching does not run on GPU yet).
